Opens from

  • Workout Builder → tap Continue (builder.html). Це перший екран In-workout — landing у pending стані
  • Clone / Repeat last → теж приземляється тут (усі сети pending з prev як орієнтиром)
  • Today banner → tap Resume якщо нічого ще не залоговано (today-in-progress.html)

Opens to

  • Tap Begin workout → live стан In-workout, cursor стає на сет 1, армується rest timer, стартує duration clock (in-workout.html) — explicit soft start
  • Tap на blank kg або reps → custom numpad (numpad.html). Збереження значення на сеті 1 = неявний soft start → теж переходить у live
  • Tap на ✓ pending-сету → лог сету = неявний soft start → live стан (in-workout.html)
  • Tap Add set → новий pending сет з копією значень попереднього (per spec §5.5, §5.9)
  • Tap на set № → set actions sheet (set-actions-sheet.html)
  • Tap top-bar ⋯ → workout actions sheet (Add exercise — додає вправу з 1 pending-сетом / Reorder / Discard, per spec §5.5)
  • Tap ← back → повертає у Builder (per spec §5.9 — нічого ще не залоговано, тренування не «почалося»)

Interactions

  • Cursor вимкнено. Жоден сет не active/next — немає amber highlight. Усі рядки в pending стилі (per spec §5.3, §5.9)
  • Soft start = тренування «починається» на першому залогованому сеті АБО при явному Begin workout — що раніше. Тоді вмикається cursor, армується rest timer, стартує duration clock
  • Setup-час не рахується у duration: планування схеми (сети/повтори, опціональні ваги) до старту — поза годинником (per spec §5.9)
  • Add set копіює значення попереднього сету (reps/weight) — рости список одним тапом
  • Вага опціональна до старту: kg blank/fillable, орієнтир — prev. Planner може пре-заповнити ваги, Improviser — ні
  • Без bottom tab bar — modal full takeover (per spec §1, §2.2)

Superset group у pending (group A)

  • Reps дочірніх вправ редагуються тут, у pending — consistent зі standalone сетами, з prev як орієнтиром (RESOLVED per redesign brief, open detail #1)
  • Один template × rounds. Кожна дочірня вправа показує ОДИН reps target (A1 8, A2 12) — це шаблон, що застосовується до всіх раундів. Колонка ×3 = template повторюється 3 рази
  • Rounds + rest живуть у superset-config-sheet (composition-level): header групи показує 3 rounds · 2:00 rest; tap ⋮ групи → superset-config-sheet.html для зміни. Per-child reps там НЕ налаштовуються
  • Немає per-child Add set у групі — кількість сетів задають rounds, не Add set. Це ключова відмінність від standalone вправ (де є Add set)
  • prev показано на кожну дочірню вправу окремо: A1 Pull-ups prev 10 × 8 (є історія), A2 Incline push-ups prev — (no-history case)
  • Вага опціональна і тут: kg blank/fillable (dashed ghost tile), орієнтир — prev, як і у standalone (per spec §5.9)
  • Візуал групи mirror'ить live-стан (in-workout-with-supersets.html): group-card = surface-1 + 3px mallard left-border, letter chip A, tinted A1/A2 на дітях (per visual §5.3). Без round-dots / active highlight — cursor вимкнено у pending

Notes

  • Один екран — два стани. Pending (цей мокап) і live (in-workout.html) — це стани одного In-workout екрана, не окремі екрани (як today-in-progress.html документує стан Today). Per spec §5.9
  • Default 1 pending set на вправу при додаванні (з Builder, quick-add chip, або mid-workout ⋯ → Add exercise). Список росте через Add set (per spec §5.9)
  • App bar: замість amber timer chip — нейтральний Not started chip (text-tertiary, не amber). Duration clock не йде — тренування ще не почалося
  • Немає rest-timer dock / floating timer — таймер не запущений у pending
  • Blank kg — faint dashed ghost tile (тапабельний → numpad), сигналізує «введи ваги, орієнтир — prev». Reps target показаний завжди
  • Вправа без історії (Incline DB press) → prev = —; reps target все одно є, вага blank
  • Обслуговує обидві персони: Planner розкладає повну схему до підйому; Improviser просто починає логати (per redesign brief)
  • Bottom bar: одна повноширинна amber CTA Begin workout (reuse cta-primary), без logged / total лічильника (нічого ще не залоговано)
  • OPEN DETAIL #2 — RESOLVED (2026-06-03): Builder-CTA = Continue → (нічого не стартує — лише веде сюди, у pending), а справжній soft-start тут лишається Begin workout. Так зникає перекриття двох start/begin-дієслів і кнопка не бреше про годинник (per redesign brief, open detail #2)
