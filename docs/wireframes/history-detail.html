Notes

  • Group rendering: per-exercise подача — як на in-workout (spec §10.3: «group rendering as in-workout»), але read-only. Кожна вправа групи — окремий під-блок: назва + її сети, пронумеровані за раундами (1/2/3), так само як standalone-вправа вище. Без roster-дубля і без round-by-round розбивки: назва вправи звучить раз — як заголовок блоку. Trade-off: губимо явний time-order (A1 r1 → A2 r1 → rest → r2…, §6.5), натомість виграємо консистентність із live-екраном і per-exercise прогресію (видно, як падали повтори в межах вправи). Round-count лишається в хедері (3 rounds)
  • Інлайн letter chip «Superset A» + 3px letter-color рейка — рейка впритул до лівого краю картки, але лише вздовж вправ (обгортка .group-children, стартує під хедером). Роздільник хедер/вправи — на всю ширину від рейки до правого краю, стикається з рейкою (еталон — in-workout, visual §5.3 + §2.5). Вправи групи йдуть блоками, розділені тонким divider; сети нумеруються 1/2/3 у стилі сет-номерів standalone (12px, text-tertiary). Літера групи A живе лише в тайтлі «Superset A», тож не дублюється на кожному рядку — інакше це шум
  • Set table = колонки SET · KG · REPS · RPE (як in-workout, але read-only — без PREV/✓). Усі колонки підписані в per-вправа хедері; одиниця названа раз там, а не на кожному рядку; ваги й повтори вирівняні (tabular-nums) → прогресія видна одним поглядом. Номер сета (або amber W для warmup, §8.3) ліворуч, RPE @N — крайня права колонка (порожня, якщо нема). Хедер і рядки відмежовані full-bleed дивайдерами — у групі від рейки до правого краю, як у standalone. Текстові лейбли Working set прибрані як шум. Однаково для standalone і дітей групи (узгоджено зі spec §10.3)
  • RPE рендериться як @8 (per visual §8.3) — text-tertiary щоб не змагатись з основними числами
  • Note block — per spec §10.3 `Workout note (якщо була залогована при completion)`. Стиль — звичайна card з NOTE uppercase label
  • Header stats — спрощений варіант 4-stats grid з spec §9.2 (там для completion screen). Тут компактний inline-row, без PR card (PR badges у v2 per spec §16). Three stats: Duration / Volume / Sets — мінімум що завжди є
  • Top-bar ⋯ прибрано (design-review F4.1): spec §10.3 явно проти actions у v1. Справа лишено 28px spacer (та сама балансувальна конвенція, що в about.html та інших sub-screens), щоб центрований заголовок не зсувався

Edge cases · related states

  • Без note — note-block просто прибирається з потоку (умовний рендер)
  • Без RPE (Show RPE OFF або сет без RPE) — суфікс @N не показується (per spec §12.2 — RPE OFF ховає бейджі, але збережені значення лишаються в db; у Browse-з-OFF — теж не показуємо, узгоджено)
  • Партіально-завершений workout: показуємо тільки залоговані сети (per spec §10.5). Не залоговані просто відсутні. У header stats Sets = логованих, не запланованих
  • Skipped exercise з сетами: відображається як звичайна вправа з тими сетами, що залогувалися; skipped marker не потрапляє у History (per spec §5.6)
  • Failed reps (0 reps): у колонці REPS стоїть 0 (колонка KG лишається, напр. 75.0) — явно, per spec §7
  • Soft-deleted custom exercise: назва і атрибути зберігаються тут через entity ID — workout не змінюється post-delete (per spec §11.9)
  • Bodyweight exercise з extra weight: формат TBD — у колонковій подачі ймовірно KG-колонка показує BW (або +10.0 для extra weight), REPS — повтори. Не відображено тут — потрібен окремий приклад
  • Кілька груп у одному workout-і — letter rotation A/B/C з different colors (visual §2.5, spec §6.3); рейка .group-children (letter-color) + інлайн letter-chip змінюються відповідно
