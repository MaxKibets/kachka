Notes

  • Letter chip + 3px left border групи — з visual.md §5.3 (Group card spec) + §2.5 (letter chip 24×24, radius 6, dark text on color)
  • Set labels: Working set / Warmup явно на кожному рядку, бо detail — це довідкова сторінка, не logging UI; читабельність важливіша за щільність. Warmup ще додатково підсилюється бейджем W у label-area (per visual §5.1, §8.3)
  • RPE рендериться як @8 (per visual §8.3) — text-tertiary щоб не змагатись з основними числами
  • Note block — per spec §10.3 `Workout note (якщо була залогована при completion)`. Стиль — звичайна card з NOTE uppercase label
  • Header stats — спрощений варіант 4-stats grid з spec §9.2 (там для completion screen). Тут компактний inline-row, без PR card (PR badges у v2 per spec §16). Three stats: Duration / Volume / Sets — мінімум що завжди є
  • Top-bar ⋯ прибрано (design-review F4.1): spec §10.3 явно проти actions у v1. Справа лишено 28px spacer (та сама балансувальна конвенція, що в about.html та інших sub-screens), щоб центрований заголовок не зсувався

Edge cases · related states

  • Без note — note-block просто прибирається з потоку (умовний рендер)
  • Без RPE (Show RPE OFF або сет без RPE) — суфікс @N не показується (per spec §12.2 — RPE OFF ховає бейджі, але збережені значення лишаються в db; у Browse-з-OFF — теж не показуємо, узгоджено)
  • Партіально-завершений workout: показуємо тільки залоговані сети (per spec §10.5). Не залоговані просто відсутні. У header stats Sets = логованих, не запланованих
  • Skipped exercise з сетами: відображається як звичайна вправа з тими сетами, що залогувалися; skipped marker не потрапляє у History (per spec §5.6)
  • Failed reps (0 reps): рядок показується як 75.0 × 0 (per spec §7 — явно)
  • Soft-deleted custom exercise: назва і атрибути зберігаються тут через entity ID — workout не змінюється post-delete (per spec §11.9)
  • Bodyweight exercise з extra weight: формат TBD (можливо BW + 10.0 × 8 або 10.0 × 8 з міткою BW). Не відображено тут — потрібен окремий приклад
  • Кілька груп у одному workout-і — letter rotation A/B/C з different colors (visual §2.5, spec §6.3); group-card border-left + letter-chip background змінюються відповідно
