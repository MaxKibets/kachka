Opens from

  • Builder → row ⋮ → Add to superset → цей combined sheet (per spec §6.2). Partner-select + rounds + rest в одному кроці — не двоступеневий picker→config
  • Builder → group ⋮ → Edit superset (edit mode — додатковий Ungroup у footer)
  • Active workout → analogous, з constraint: усі candidates мають 0 залогованих сетів (per spec §6.1)

Opens to

  • Tap Save → group створено / оновлено → закриває sheet → Builder з builder-with-supersets.html
  • Tap Cancel → закриває sheet без змін → повертається на Builder
  • Tap Ungroup (edit mode тільки) → confirmation sheet → Розпадається на флет вправи (per spec §6.7)
  • Свайп вниз = Cancel (per spec §1)

Interactions

  • Multi-select checkboxes — мінімум 2, максимум 5 (per spec §6.1). Save disabled якщо selected < 2
  • Disabled candidates показані з reason inline (Already started якщо є залоговані сети, In another superset якщо вже в групі — per spec §6.2)
  • Rounds stepper: 2–10, default 3. Кнопка − disabled на 2, кнопка + disabled на 10
  • Rest presets: 60s / 90s / 120s / 180s, default 90s. Single-select chip-стилі
  • Свайп вниз або тап на scrim — dismiss = Cancel
  • Backdrop затемнений (scrim + builder opacity 0.4) — фокус на sheet

Notes

  • Sheet title Configure superset A — letter уже відомий бо літери присвоюються при створенні (per spec §6.3)
  • Subtitle expects 2–5 exercises explicit (per spec §6.1) — щоб юзер бачив constraint без помилок
  • Footer order top-to-bottom: Cancel (ghost) → Save (primary amber CTA) → Ungroup (destructive, edit mode тільки). Per spec §1 — Cancel вгорі, destructive внизу. Save як primary action — between them
  • Edit mode додає Ungroup у footer (per prompt). Create mode (нова група) — без цієї кнопки
  • Assumption: Per spec §6.2 у create flow є окремий партнер-picker screen перед config sheet. У цьому wireframe показано combined sheet — multi-select picker + config в одному. Це reasonable simplification: для 2-5 candidates з тим самим контекстом окремий screen виглядає overkill, sheet вміщає все. Якщо юзер хоче двоступеневий flow — можна додати в наступну ітерацію
  • Assumption: Rounds picker = stepper (per prompt дозволяє "numeric stepper або dropdown"); ставлю stepper бо швидший для thumb (один тап = ±1)
  • Assumption: Rest picker = preset chips (per prompt). Якщо потрібно custom value — TBD у impl (можна додати Custom… chip який відкриває numpad)
  • Грабхендл — 36×4 pill з rgba(white, 0.16) у верхньому центрі, стандартний iOS/Android indicator
  • Sheet visual — surface-1 background з 18px top corner radius, top border subtle. Drop-shadow для відриву від backdrop

Edge cases · related states

  • Create mode (без Ungroup) — той самий sheet без destructive button
  • Active workout edit mode — додатковий constraint: rounds можна збільшити завжди, зменшити тільки до значення ≥ current completed rounds (per spec §6.7) — disabled state на − якщо вже на min
  • Якщо selected = 0 (юзер uncheck-нув все що вибрав) — Save disabled, message Pick at least 2 exercises
