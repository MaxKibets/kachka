Opens from

  • Workout Builder → + Add exercise (builder.html)
  • Active workout → top bar ⋯ → Add exercise (in-workout — TBD у Batch 3)
  • Active workout → per-exercise ⋮ → Insert after (per spec §11.1)

Opens to

  • Tap row → returns selected exercise to caller (closes modal). Builder додає 3×8, Active додає 3×8 і ставить cursor
  • Tap + Create custom → custom creation sheet (exercise-create.html — TBD у Batch 5) з порожнім name
  • Tap × → closes modal без selection, повертається на caller
  • Search results empty → inline Create as 'X' affordance (no-results state, per spec §11.6) — окремий wireframe TBD; sticky + Create custom завжди доступний

Interactions

  • Sticky search input + filter chips під ним. Скролиться тільки список
  • Filter chips — single-select (один або жоден). Default — none selected (показує весь каталог)
  • Filter scroll горизонтально якщо chips не вміщаються
  • Search — case-insensitive substring (per spec §11.3). AND-композиція з muscle filter
  • Search/filter — reset кожен раз як відкривається picker (per spec §11.3, §11.4)

Notes

  • Header: close × зліва (не back chevron) щоб підкреслити modal-нічність — це не push, це modal який можна dismiss-нути
  • Single-select Add mode у v1 (multi-select → v1.x per spec §11.10)
  • Bodyweight показано як inline-tag bodyweight в muscle list — мінімалістично, без окремого badge
  • System exercises у Add mode — без ⋮, без chevron, без custom badge (read-only, тап на рядку селектить)
  • Assumption: Spec §11 каже "Sticky `All` chip first, selected by default", але prompt каже "Default state — none selected, 7 chips single-select". Дотримуюсь prompt-у — без явного All chip-у; якщо нічого не selected, показується весь каталог
  • Assumption: Sample list з 13 вправ — представляє типове розмаїття (compound + isolation + bodyweight). Реальний seed буде ширшим

Edge cases · related states

  • Browse mode (з Profile/DATA) — exercise-picker-browse.html
  • Empty search results з + Create as 'X' — окремий state, TBD якщо потрібно
  • Active workout context: вправи що мають залоговані сети у поточному workout-і не disabled — picker дозволяє додати ту ж вправу повторно (свідомо, юзер може хотіти додати її після іншої)
