  • Tap Cancel або swipe down → закриває sheet, повертає на In-workout без змін

  • Backdrop scrim ~60% dim per task spec; tap по scrim також = Cancel (consistent з action sheet pattern)

Notes

  • Цей sheet — лайтова версія finish-flow для v1: показує summary + бере опц. note, без full completion screen з 4 stats cards описаним у spec §9.2. Reasonable default: spec §9.2 описує completion screen з PR card, exercise summary collapsible, etc. — це буде окремий екран, не sheet. Цей finish-sheet.html — це варіант "tap Finish → quick confirm sheet → save". Альтернативний інтерпретейшн — full screen — TBD з юзером
  • CTA — amber primary, узгоджено з visual §5.4 (primary CTA = amber). Mallard уже несе success/done semantics; кнопка тут конструктивна, не destructive — amber правильно
  • Sets logged показує 12 of 12 з зеленим mallard ✓ pill (all done). Якщо partial — заміна на amber dot + текст без ✓ (див. edge cases). Окремий рядок Skipped exercises з'являється тільки коли є skipped (умовний показ — виключений у цьому "happy path" mockup-і)
  • Per spec §9.1 partial-completed дозволено зберігати (`Done? No → Manual finish → Confirm save partial → Summary`). Тут показано all-done. У partial state потрібен extra warning рядок — TBD стиль (інлайн hint під summary?)
  • Workout note placeholder як inline textarea — згідно task spec (`note-on-workout TBD; зробити inline якщо легко`). Якщо у impl потрібна повна textarea з expanded state — spec §9.2 згадує її серед компонентів completion screen. Тут — мінімальна, 1-row resize:none
  • Behind layer (faint preview In-workout) декоративний: показує що це поверх а не повний takeover. Реальний impl бере справжній snapshot активного workout-а

Edge cases · related states

  • Partial complete: Sets logged рядок показує amber dot замість зеленого ✓: ● 12 of 14. Додатковий info-рядок: 2 sets pending — they won't be saved у text-secondary, font-size 12. Per spec §9.1 — підтверджено partial save
  • Skipped exercises: додатковий summary-row: Skipped · [skipped-tag]1 skipped[/skipped-tag]. Per spec §5.6 — skipped marker зберігається у структурі, у History показується факт сетів. У finish-summary показуємо явно щоб юзер не пропустив
  • 0 sets logged (юзер просто стартував і вирішив фінішувати): треба окремий confirm — `Save empty workout?` з warning. Можливо краще зразу запропонувати Discard. Open question
  • Discard flow: окрема destructive sheet через top-bar ⋯ → Discard workout → confirmation per visual + spec §1 (Cancel вгорі, Discard внизу). Не показано тут — окремий wireframe TBD
  • Full completion screen (spec §9.2 — 4 stats cards, PR card, collapsible exercise summary) — окрема еволюція, не цей sheet. Якщо юзер хоче саме full screen, тоді цей файл стає intermediate confirm sheet перед цим екраном. TBD з юзером
