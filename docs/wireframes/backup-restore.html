Backup & restore

Export
Export your workouts and exercises as a JSON file you can share or save elsewhere.
Import
Import a previously exported JSON file. You'll see a preview before anything is changed.
Auto-backup and encryption — in v2.
Your data stays on this device.

Opens from

  • Profile (profile.html) → DATA → Backup & restore (push на Profile stack, per spec §12.4)

Opens to

  • Tap Export data → app серіалізує всі юзерські дані в JSON → native share sheet (Files / iCloud / AirDrop / Drive / email / etc., per spec §13.2). Toast Backup exported на success
  • Tap Import file → native file picker (filter .json) → validation → backup-import-preview.html
  • Tap ← back → profile.html

Interactions

  • Дві симетричні зони — Export і Import — кожна як card з icon + description + button
  • Без Last export indicator-у (per spec §13.1 — manual режим без напоминання)
  • Export — single tap → native share sheet (без свого destination picker per spec §13.2)

Notes

  • Без tab bar (sub-screen на Profile stack)
  • Кнопки secondary style — Mallard outline, не amber primary CTA. Чому: backup це не destination action в звичному flow (як Start workout), це utility tool. Mallard = data-related action (узгоджено з DATA section в Profile)
  • Іконки: upload (стрілка вгору з box) для Export, download (стрілка вниз з box) для Import. У Mallard tint surface на 36px square
  • Footer note підкреслює v2-обмеження і local-only архітектуру (per spec §13.7 + tech privacy stance)
  • Description copy — на одне речення, узгоджено з spec §13.1 але переписано від першого користувача ("Export your workouts and exercises..." замість "Save all your data...")
  • Plain JSON у v1, без encryption (per spec §13.5)

Edge cases · related states

  • Successful export → toast Backup exported, лишається на цьому screen-і
  • Import validation fails → error sheet з конкретною причиною (per spec §13.6 — corrupted, not Kachka backup, newer schema, migration fail)
  • Older schema → auto-migrate JSON before preview (silent, per spec §13.6)
  • Newer schema → error sheet This backup was created with a newer version of Kachka. Update the app to import. (per spec §13.6)
