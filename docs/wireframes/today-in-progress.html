Opens from

  • App launch коли є збережений in-progress workout (crash recovery, юзер закрив додаток мід-tworkout)
  • Tab bar → Today під час паузи в активному тренуванні (юзер закрив modal Active workout не завершивши)

Opens to

  • Tap Resume → Active workout modal на збереженому state (in-workout.html — TBD у Batch 3)
  • Tap Discard → bottom sheet confirmation (Discard workout?) → видаляє in-progress, banner зникає, екран стає today-has-history.html
  • Repeat last / Choose from history / Start blank workout — disabled (dimmed, non-tappable) поки висить banner. Per spec §3.1.c єдиний шлях далі — Resume або Discard. Constraint "one active workout at a time"; навмисно НЕ confirm-діалог — banner уже несе Resume/Discard
  • Tap ⋯ → top-bar action sheet: Exercise database / Settings

Interactions

  • Banner НЕ swipe-dismissable — навмисно sticky щоб юзер не "загубив" workout
  • Свідомо NOT auto-resume per spec §3.1.c — захист від попадання в забуте старе тренування

Notes

  • Banner-стан показуємо в стилі mallard left-border (як group card в visual §5.3) — саме mallard сигналізує "позитивна важливість"
  • Resume = filled mallard, Discard = ghost. Discard відкриває confirmation bottom sheet (per spec §3.1.c + §1 confirmation pattern)
  • Open question: чи показувати progress підрахунок (3 of 8) одразу чи лише назву. Поки ставлю — корисно перед Resume

Edge cases · related states

  • Якщо юзер discard-нув — falls back to today-has-history.html
  • Якщо у юзера немає history (вперше запустив, почав тренування, закрив) — banner все одно показується, під ним today-first.html empty state без card
