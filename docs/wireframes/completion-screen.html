Opens from

  • in-workout.html → tap Finish workout у top-bar ⋯ (manual, partial allowed) — per spec §9.1
  • in-workout.html → auto-prompt banner коли всі сети залоговані → tap Finish
  • Не досяжний при 0 залогованих сетів — той шлях йде одразу в Discard confirm, completion screen пропускається (spec §9.1)

Opens to

  • Tap Save to history → workout зберігається → history-list.html (свіжий entry зверху). Per spec §9.1 → §10.5
  • Tap Discard workout → destructive confirm sheet (Cancel вгорі, Discard внизу, per §1) → confirm видаляє workout, не зберігає; cancel повертає на цей екран. Окремий confirm wireframe — TBD
  • Немає back: екран термінальний, вийти лише через Save або Discard (per spec §1 — In-workout modal закривається тільки свідомою дією)

Interactions

  • Vertical scroll у тілі; action bar (Save / Discard) — fixed внизу, поза скролом
  • Tap на note input → системна клавіатура; note редагується тут до Save (per spec §9.2 — textarea, опціональна)
  • Exercise summary — collapsible (details), default expanded. Tap на header → згорнути/розгорнути, chevron обертається

Notes

  • Повноекранний, не bottom sheet — per spec §9.2 (рішення зафіксоване: один крок Finish → цей екран → Save/Discard, без проміжного quick-confirm sheet). Замінив видалену lightweight-ітерацію finish-sheet.html
  • Stats grid 2×2, 4 картки per spec §9.2: Volume / Duration / Sets / Personal records. Перші три — нейтральні surface-1; Personal records — info-accent
  • PR card = info-accent (dusty blue --letter-a #5C7B95) per spec §9.2 "PR card візуально виділена info-кольором — єдиний motivational accent". Вибір кольору: visual §2.4 мапить info → dusty blue (конфлікт з letter A відмічений там як TBD; тут беремо як reasonable default до lock semantic palette). PR-вправи перелічені під лічильником
  • PR marker ◆ біля назви вправи в summary і на stats card — per spec §9.3
  • Primary CTA = amber Save to history (visual §5.4 — primary CTA amber; mallard несе success/done, але кнопка тут конструктивна-нейтральна)
  • Discard = rust text --letter-e #B85842 — secondary text button per spec §9.2; rust як destructive default (узгоджено з row-menu/INDEX semantic-default)
  • Group rendering у summary: компактний — інлайн чіп Superset A + 3 rounds meta + mini-list вправ у порядку (без per-row ординалів). Повна round-by-round розбивка (вправи в раунді нумеровані 1/2, літера групи — лише в тайтлі) — у History detail (§10.3), не тут (summary = per-exercise counts per spec §9.2)
  • App-bar — лише centered title Workout complete, без back / ⋯ (термінальний екран)

Edge cases · related states

  • Partial-completed (юзер вручну Finish, не всі сети): дозволено зберігати (spec §9.1). Sets card показує залоговані (напр. 12), не заплановані; опц. підрядок-hint 2 sets not logged — won't be saved у text-secondary. Volume/PR рахуються лише по залогованих
  • 0 PR: Personal records card → нейтральний surface-1 (без info-accent), value 0, без pr-names, без ◆ у summary. Motivational accent зникає коли нема чого святкувати
  • Без note: textarea лишається порожня з placeholder; при Save note не зберігається (умовно)
  • Skipped exercise з сетами: показується як звичайна вправа з тими сетами що залогувались; skipped marker не йде в History (spec §5.6)
  • 0 sets logged: цей екран НЕ показується — Finish веде одразу в Discard confirm (spec §9.1). Тут не моделюється свідомо
  • Кілька груп: letter rotation A/B/C, інлайн letter-chip колір змінюється відповідно (Superset B, Superset C, visual §2.5)
