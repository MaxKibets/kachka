Opens from

  • Pending (pre-start) стан того ж екрана — In-workout відкривається спершу як pending (in-workout-pending.html, per spec §5.9). Цей мокап — live стан після soft start
  • Workout Builder → tap Continue (builder.html — TBD у Batch 2). Веде у pending стан, не одразу сюди
  • Today banner → tap Resume (today-in-progress.html)

Opens to

  • Tap on set kg / reps cell → custom numpad sheet (numpad.html)
  • Tap on set number → set actions sheet (set-actions-sheet.html)
  • Tap on ✓ → save set + advance cursor + start rest timer (rest-timer.html)
  • Tap the filled note icon → toggle the exercise hint banner (expand/collapse); tap the banner ✎ → note editor (exercise-note-sheet.html). Якщо нотатки нема — note-іконки нема, додати через ⋮ → Add note
  • Tap exercise ⋮ → exercise actions sheet (TBD — Insert after / Skip / Remove / Add to superset / Add note)
  • Tap top-bar ⋮ → workout actions sheet (TBD — Add exercise / Reorder / Discard)
  • Tap ← back → закриває modal без discard, банер на Today (today-in-progress.html)
  • Hold Finish (press-and-hold ~600 мс, не тап) → повноекранний completion-screen.html (≥1 set logged; 0 sets → одразу Discard confirm per spec §9.1, §9.2)
  • Cursor exits viewport → return chip (pull-to-cursor.html)

Interactions

  • Vertical scroll. Auto-scroll keeps active set in view; manual scroll override pauses auto-scroll (per spec §5.8)
  • Tap ✓ on active set = save + advance cursor + start rest. Closing last set of last exercise = banner suggesting Finish workout
  • Long-press на set row = secondary trigger для set actions (per spec §8.1)
  • Active set: kg/reps поля тапабельні незалежно (numpad перемикає focus між полями)
  • Без bottom tab bar — modal full takeover (per spec §1, §2.2)
  • Soft start: цей live стан настає коли тренування «почалося» — на першому залогованому сеті АБО після явного Begin у pending стані (що раніше). Тоді вмикається cursor, армується rest timer, стартує duration clock (per spec §5.9). До того — pending стан
  • Add set копіює значення попереднього сету (reps/weight) — додати сет це один тап (per spec §5.5, §5.9)

Edge cases · related states

  • Pending (pre-start) стан — in-workout-pending.html (стан перед soft start, per spec §5.9)
  • З суперсетами — in-workout-with-supersets.html
  • Numpad open поверх → numpad.html
  • Set actions sheet → set-actions-sheet.html
  • Rest timer running → rest-timer.html
  • Cursor поза viewport → pull-to-cursor.html
  • Skipped exercise — TBD визуально (per spec §5.6: лишається в структурі помічена Skipped)
  • Failed reps (0) — TBD візуально (per spec §5.7 + visual §5.1)
  • Завершена вправа — collapsible до one-line summary (per spec §5.4) — TBD detail у наступних мокапах

Notes

  • Header — назва + edit-pencil + meta-рядок під нею + amber timer chip + ⋮. Timer chip — workout elapsed (per visual §5.6)
  • Exercise note (author hint) — inline-банер під назвою (surface-2, текст + ✎ edit). Дефолт згорнуто; filled note-іконка в хедері = є нотатка, tap розгортає/згортає (тут показано розгорнутим для демо). Вправа без нотатки (напр. Tricep pushdown) — без note-іконки, додавання через ⋮ (per spec §5.2, builder §4.4). Workout-level author note (program-format §139) — display у v2
  • Set row grid — 32px | 1fr | 78px | 60px | 40px з 8px gaps, 14px padding (per visual §4.1)
  • Done state — opacity 0.55, mallard ✓ circle. Active state — top/bottom amber borders, surface-2 фон, amber empty circle; kg/reps — пласкі (без фону, розмір як у таблиці), але amber-кольором як підказка «редагується». Pending — text.tertiary values, 1px subtle empty circle (per visual §5.1)
  • Warm-up set — у колонці SET замість номера стоїть amber-гліф W (per spec §8.3). Рядок логується як звичайний (вага×повтори, ✓), але не йде у volume/PR; робочі сети нумеруються 1, 2, 3 без урахування розминок. Тип перемикається в set-actions sheet (set-actions-sheet.html, segment Working | Warm-up)
  • Active row "ковтає" border-divider зверху і знизу через negative margin — щоб не було подвійних ліній (subtle divider + amber border)
  • Bottom action bar: logged / total ліворуч (Plex Mono, secondary text, Plex Mono numbers primary), amber CTA повна ширина праворуч (per brief)
  • Hold to finish — завершення = press-and-hold (~600 мс) з fill-індикатором, не одиночний тап (per spec §9.1). Причини: flow-ending дія раз на тренування + idle-бар відвойовує той самий слот, де в rest-режимі сидить Skip (§5.10), тож холд поглинає випадковий/подвійний тап одразу після Skip. Прецедент — Strava / Nike Run Club «hold to finish». Ранній відпуск скасовує без ефекту
  • "Add set" — secondary button per visual §5.4 (dashed border, transparent bg). Копіює попередній сет (per spec §5.9)
  • Перед цим екраном — pending (pre-start) стан (in-workout-pending.html): усі сети pending, cursor вимкнено, timer/clock не йдуть, час підготовки не рахується в duration. Soft start (перший лог АБО Begin) переводить екран у цей live стан (per spec §5.9)
