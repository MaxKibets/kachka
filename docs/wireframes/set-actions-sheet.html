Opens from

  • In-workout · tap на номер сета (primary trigger per spec §8.1) — будь-який рядок (done / active / pending) (in-workout.html)
  • In-workout · long-press на set row (secondary trigger per spec §8.1)

Opens to

  • Tap Working / Warm-up у сегменті Set type → комітить тип сета, sheet закривається. Warm-up → бейдж W біля номера + виключення з volume/PR; Working → сет повертає порядковий номер (per spec §8.2 / §8.3)
  • Tap RPE value → коміт RPE на сет, sheet закривається. Бейдж @8 у рядку
  • Tap Delete set → bottom sheet confirmation (per spec §1: Cancel вгорі + destructive внизу). Підтвердження → видалення сета
  • Swipe down / tap на dim → закриває sheet без змін

Interactions

  • Inline RPE picker — 9 кнопок (6, 6.5, 7, 7.5, 8, 8.5, 9, 9.5, 10), single-select. Tap одразу комітить значення. Selected — mallard-dim фон з mallard border
  • Якщо RPE вже встановлено — selected кнопка вже виділена. Tap іншої = переписати. Tap selected ще раз = remove (TBD)
  • Якщо Show RPE у Settings вимкнено (per spec §12.2) — рядок RPE і inline picker не показуються
  • Set type — inline сегмент Working | Warm-up, single-select, tap одразу комітить (як RPE-пікер). Поточний тип підсвічений (mallard-dim) — обидва стани завжди видимі, тож revert очевидний: випадковий warmup → тап Working повертає робочий статус і перенумеровує сети

Edge cases · related states

  • Сет ще не залогований (pending/active) — Delete доступний, прибирає сет з вправи. Якщо сет вже done — Delete теж доступний, але треба явне confirm (per spec §8 + §5.5)
  • Last set у вправі — Delete vs add: видалити можна; якщо вправа лишилась з 0 сетами — TBD (видалити вправу? показати empty стан?)
  • RPE off у settings — рядок прихований (per spec §12.2)
  • Warmup сет — у volume і PR не йде (per spec §8.2)

Notes

  • Sheet — bottom-anchored, повна ширина, grab handle вгорі (per visual §5.7 + spec §1)
  • Header label — Plex Mono number у контексті Bench press · Set 3
  • RPE inline (а не submenu): rare action але важливий для тих хто RPE юзає; краще одразу tap-to-commit без додаткових екранів
  • Range RPE 6 — 10 свідомо (per common gym usage — нижче 6 малоінформативно). 0.5 step як в загальній практиці. Якщо юзер запросить 1 — додамо в v1.x
  • Destructive Delete — у нашій мові #B85842 (letter E rust). Resolves visual §2.4 TBD: для destructive перевикористовуємо rust-letter-color як warning/danger у v1 (semantic colors TBD у visual §2.4)
  • Set type = сегмент, а не toggle (design-review, supersedes warmup-toggle): старий toggle ховав поточний стан і не давав явного шляху назад. Сегмент показує обидва типи завжди, revert у один тап, консистентно з inline-RPE й моделлю «set type» конкурентів (Hevy / Strong)
  • Section divider — тонкий hairline (0.5px var(--border-divider)) + відступ: відділяє кожен inline-блок (Set type · RPE) один від одного та від destructive Delete, у стилі sheet-сімейства (per visual §5.7)
  • Open question: показувати rest timer override як action (start rest now / skip rest)? Поки не включаю — rest auto-start після ✓ tap, окремий action sheet не потрібен
