Opens from

  • Edit існуючої custom — той самий sheet з prefilled values (per spec §11.8)

Opens to

  • Tap Save → нова custom вправа в db, modal закривається. Якщо викликалось з Add mode — вправа автоматично селектиться і повертається в caller (per spec §11.7)
  • Tap × (header) / свайп вниз / scrim → закриває без збереження; якщо форма змінена — confirm Discard changes? (per spec §1)

Interactions

  • Tap на Name field → системна клавіатура (text input)
  • Tap на muscle chip → single-select. Активний chip — Mallard tint
  • Tap на Bodyweight toggle → switches state, не блокує save
  • Save disabled поки Name empty (тут показано як neutral state, без error indicator поки юзер ще не намагався save)
  • Dismiss форми — header × / свайп вниз / тап scrim (form sheet, per visual §5.7)

Notes

  • Form bottom sheet (не повноекранний sub-screen): grab-handle + заокруглений верх + scrim із dimmed parent — той самий sheet-вигляд, що й superset-config (per visual §5.7). Тому header має × (cancel/close), не ←
  • Чому Legs preselected у demo: для візуального showcase активного chip-state. Реальний impl — нічого не selected на open
  • Validation: name unique (case-insensitive) per spec §11.7. У neutral state (поки юзер тільки зайшов або ще не save-ив) — без error indicator. Після Save з duplicate-name — inline error під input-ом (TBD як окремий error-state мокап)
  • Bodyweight toggle з description під label-ом — пояснює effect (kg field ховається у workout per spec §7.4)
  • Save button — primary CTA з checkmark icon (per visual §5.4 Primary); окремої footer-кнопки Cancel немає — cancel це header × (unify-on-× decision, як у superset-config)
  • Поля у v1: name, muscle group, isBodyweight (per spec §11.7). Notes / equipment — v1.x (відкладено per spec §11.10)

Edge cases · related states

  • Empty name + tap Save → inline error Name is required (red-tinted, TBD style — visual §2.4 semantic)
  • Duplicate name → inline error 'X' already exists
  • Edit existing custom → той самий screen з title Edit custom exercise + prefilled values (per spec §11.8)
  • Викликано з no-results affordance → name prefilled з search query (per spec §11.6)
