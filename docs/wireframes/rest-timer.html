Opens from

  • In-workout · tap ✓ on active set (auto-start rest, per spec §5.2 + §6.6)
  • В group context — після завершення останньої вправи раунду (per spec §6.5)

Opens to

  • Timer expires (00:00) → idle state, haptic pulse + опц. sound (per spec §12.3)
  • Tap Skip → idle, advance cursor immediately
  • Tap +15s / −15s → adjust countdown without dismissing
  • Tap на новий active set → rest продовжується paralel (юзер може почати наступний сет до закінчення rest); або dismiss timer (TBD)

Interactions

  • Ring progress = mallard, fills counter-clockwise as time elapses (started full → empty at 0:00)
  • Mallard pulse-dot під часом — subtle indicator що Rest haptic ON (per visual §7 + spec §12.3). Якщо Rest haptic OFF — pulse-dot прихований
  • Floating timer перекриває нижню частину scroll list — context active set частково схований, але scroll все одно доступний (можна прокрутити timer повз)
  • Drag-to-collapse / dismiss — TBD у v1.x; зараз tap Skip або wait для closure
  • Per spec §6.6 state machine: Idle → Rest → Idle. Rest can be extended by tap +15s

Edge cases · related states

  • Group rest — label у center має letter-color tinted: A · Rest з amber-pulse (per spec §6.6: A · Rest 2:00). Не показано в цьому мокапі, але імплементація — letter chip ліворуч від Rest
  • Rest haptic OFF + sound OFF — pulse-dot прихований, тільки візуальний countdown (per spec §12.3)
  • Timer expired — ring повний green, time = 00:00, haptic + опц. sound, dock автоматично ховається через ~1.5s
  • Юзер skipped → cursor advances, dock disappears
  • Two timers одночасно (workout elapsed + rest) — workout chip у header лишається тікати; rest — окремий floating circle. Конфлікт відсутній

Notes

  • Reasonable default per brief — spec не дає точного візуалу. Floating circular dock над bottom action bar обрано (а не bottom-anchored bar) тому що: (1) не блокує існуючий action bar, (2) circular ring = очевидний countdown affordance, (3) pulse-dot під часом дає channel для rest haptic indicator
  • Альтернатива (не показана) — full bottom dock над action bar з прямою прогресбаром і inline кнопками. Простіша, але "з'їдає" вертикальний простір; у Hevy/Strong використовується саме circular floating, для нашої density-orientованої системи теж працює краще
  • Color: ring у mallard — primary success, не amber (бо amber вже = active focus у set row). Pulse-dot теж mallard для consistency
  • Time format: MM:SS, Plex Mono Medium 30pt — найбільший number в системі (per visual §3.3 number large), бо це primary signal "коли наступний сет"
  • Open question: чи треба Pause? У типовому use-case rest не паузиться (юзер скаже Skip). Поки опускаю; додамо за запитом
  • Open question: позиція floating timer — чи overlap зі scroll або вище. Поки 92px from bottom (above action bar). На малих екранах може треба зменшити circle до 110px
