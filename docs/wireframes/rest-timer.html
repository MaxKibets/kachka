Opens from

  • In-workout · tap ✓ on active set (auto-start rest, per spec §5.2 + §6.6)
  • В group context — після завершення останньої вправи раунду (per spec §6.5)

Opens to

  • Timer expires (00:00) → idle state, haptic pulse + опц. sound (per spec §12.3)
  • Tap Skip → idle, advance cursor immediately
  • Tap +15s / −15s → adjust countdown without dismissing
  • Tap на новий active set → rest продовжується paralel (юзер може почати наступний сет до закінчення rest); або dismiss timer (TBD)

Interactions

  • Rest = bottom bar's rest mode (§5.1): accented countdown з великим mono MM:SS + тонка progress-смуга (mallard) на всю ширину бару, depletes as time elapses
  • Mallard pulse-dot біля лейбла Rest — subtle indicator що Rest haptic ON (visual §7 + spec §12.3). Якщо Rest haptic OFF — pulse-dot прихований
  • Не плаває над списком — сети не перекриті; бар фіксований унизу як завжди
  • Coexists з return-to-cursor chip (§5.8): chip плаває над баром, rest — у барі. Різні шари, без колізії — обидва можуть бути видимі одночасно
  • Per spec §6.6 state machine: Idle → Rest → Idle. Rest can be extended by tap +15s, або Skip → advance

Edge cases · related states

  • Group rest — лейбл зліва letter-color tinted: A · Rest (spec §6.3/§6.6: A · Rest 2:00). Standalone = просто Rest. Letter chip префіксує Rest у барі
  • Rest haptic OFF + sound OFF — pulse-dot прихований, тільки візуальний countdown (per spec §12.3)
  • Timer expired — 00:00, haptic + опц. sound, бар повертається в idle mode (progress + Finish, як у in-workout.html) через ~1.5s
  • Юзер skipped → cursor advances, бар → idle
  • Two timers одночасно (workout elapsed + rest) — workout chip у header тікає; rest — у bottom bar. Різні зони, конфлікт відсутній

Notes

  • Decision (design-review F3.1): rest = accented bottom-bar countdown, не плаваюче коло. Узгоджено зі spec §5.1/§6.3/§6.6 (bottom bar несе rest state) і прибирає колізію з return-to-cursor chip (§5.8), який ділив той самий 92px-слот
  • Rejected: велике floating circular ring. Glanceable здалеку, але ~180px висоти, штовхає список сетів, і колізить з chip. Hybrid bottom bar (великий mono number + full-width progress) лишається glanceable без окремого шару
  • Color: progress-смуга у mallard — primary success, не amber (amber = active focus у set row). Pulse-dot теж mallard
  • Time format: MM:SS, Plex Mono Medium ~26pt — великий number, бо це primary signal "коли наступний сет"
  • Open question: чи треба Pause? У типовому use-case rest не паузиться (юзер скаже Skip). Поки опускаю; додамо за запитом
