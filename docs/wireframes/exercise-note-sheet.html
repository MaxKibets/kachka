Note
Author hint — form cue, tempo, reminder. Rendered read-only in the active workout.

Opens from

  • Builder · row ⋮ → Add note / Edit note (builder-row-menu-sheet.html, per spec §4.4)
  • Active workout · hint-банер ✎ під назвою вправи (edit існуючої нотатки) (in-workout.html, per spec §5.2)
  • Active workout · exercise ⋮ actions sheet → Add note / Edit note (TBD sheet, per spec §5.5)

Opens to

  • Tap Save → зберігає текст у exercise.notes, sheet закривається, повертає в caller (Builder або Active workout)
  • Tap Clear → очищає textarea (тільки поле, не комітить). Далі Save з порожнім полем → нотатка видаляється — без окремого destructive/confirm
  • Tap × / свайп вниз / scrim → закриває без збереження; якщо текст змінено — confirm Discard changes? (per spec §1)

Interactions

  • Tap у textarea → системна клавіатура (free text, multi-line). Sheet піднімається над клавіатурою (per visual §5.7)
  • Dismiss форми — header × / свайп вниз / тап scrim (form sheet, per visual §5.7)
  • Без стека sheet'ів: коли вхід — меню-sheet (row ⋮ у Builder / exercise ⋮ в Active), воно закривається першим, а вже потім editor підіймається над екраном (Builder / Active workout), не над меню. Scrim дімить екран, не меню; закриття editor повертає на екран, а не в меню (per spec §2.5 — sheets never stack). Вхід через ✎ hint-банер — прямо з екрана, без меню
  • Mode-aware: порожня нотатка → title Add note, Clear прихований, Save disabled (нема чого зберігати). Існуюча нотатка → title Edit note + видимий Clear (показано тут)

Notes

  • Один shared-компонент на всі 3 входи (Builder row menu, Active note icon, Active exercise menu) — усі редагують одне поле exercise.notes (program-format §6). Backdrop тут — Builder, бо це первинний вхід; з Active workout той самий sheet поверх in-workout
  • Form bottom sheet: grab-handle + заокруглений верх + scrim із dimmed parent — як exercise-create / superset-config (per visual §5.7). Тому header має × (cancel/close), не ←
  • Видалення нотатки (design-review, supersedes "two paths"): окремого destructive Remove note немає — Clear біля поля очищає textarea, а Save з порожнім полем прибирає нотатку. Закриття без змін — header × (вже є). Editing optional-поля + Save = природна модель; destructive-рядок + його confirm були зайві
  • Це author hint рівня вправи, не per-set. Рендериться read-only в Active workout (per spec §4.4 + §5.5). Per-set notes у v1 немає (program-format §11 frozen — 3 levels: program / workout / exercise)
  • Save — primary CTA з checkmark (per visual §5.4); окремої footer-кнопки Cancel немає — cancel це header × (unify-on-× decision, як у exercise-create)
  • Без жорсткого ліміту символів у v1; textarea росте, лічильник не показуємо

Edge cases · related states

  • Add mode (нова нотатка): title Add note, textarea порожній з placeholder, Clear прихований. Save disabled поки текст порожній (нема чого зберігати)
  • Вправа в суперсеті: нотатка на конкретній child-вправі групи (program-format §6 group-exercise notes) — той самий редактор
  • Дуже довгий текст → textarea скролиться в межах sheet-body; sheet не перевищує 82% висоти
  • Author note з імпортованої програми → той самий редактор показує prefilled текст (Edit mode)
