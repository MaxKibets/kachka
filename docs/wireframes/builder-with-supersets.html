Opens from

  • Той самий screen що builder.html — варіант з суперсетом створеним юзером
  • Tap Add to superset у row menu (builder-row-menu-sheet.html) → партнер picker → superset-config-sheet.html → group створено

Opens to

  • Tap group ⋮ (у group header) → group action sheet (TBD у Batch 3 — patterns в builder-row-menu-sheet і set-actions-sheet): Edit rounds/rest, Add exercise to group, Remove exercise from group, Reorder inside (drag), Ungroup (per spec §4.5)
  • Tap row ⋮ на A1/A2 → row action sheet з контекстним Add to superset = inactive (вже у групі), додатково Remove from group
  • Tap row ⋮ на Bench press / Tricep pushdown → builder-row-menu-sheet.html з Add to superset · A (індикатор що існує група A — додає до неї)
  • Tap Continue → In-workout у pending state (pre-start). Сети/повторення — у тому числі reps для group children — налаштовуються там, не в Builder (per spec §4.3 + §5.9). Live-стан: in-workout-with-supersets.html

Interactions

  • Drag handle на group header (зовнішній лівий край) — рухає всю групу як одне ціле (per spec §4.6)
  • Drag handle всередині group rows — reorder тільки в межах групи (per spec §4.5). Хендл дітей — на тій самій відстані від рейки, що й хендл хедера від краю картки; вкладеність несе сама рейка, тож дитину не сплутати з хедером
  • Group card: surface-1 + 0.5px subtle border з усіх боків. 3px mallard-рейка — впритул до лівого краю картки, але лише вздовж вправ (стартує під хедером). Роздільник хедер/вправи — верхній бордер першого рядка, на всю ширину від рейки до правого краю, стикається з рейкою (еталон — in-workout, per visual §5.3)
  • Letter chip A — маленький інлайн-чіп на рядку тайтлу, читається Superset A (height 16, radius 6, color --letter-a dusty blue, text #0A0A0A). Refinement visual §2.5

Notes

  • Letter chip A лишається лише на хедері, інлайн у тайтлі (Superset A). Children-вправи без per-row літери — порядок задає позиція в картці. Крос-реф A1/A2 лишається у History та компактних текстових лейблах (rest A · Rest, return-to-cursor A · Set 3), де немає рамки картки (refinement visual §2.5)
  • Group header subtitle: Superset · 3 rounds · 90s rest — composition-level, лишається в Builder. У builder поки немає progress, у Active workout буде round X of Y (per spec §6.4)
  • Builder cards (standalone і group children) показують м'язові групи, не reps. Reps для group children налаштовуються в In-workout pending state, разом зі standalone сетами (per spec §4.3 + §5.9, resolved open detail #1). Groups не мають per-child Add set — кількість задають rounds
  • Кількість раундів керується group-level rounds (тут 3) — per round = 1 set кожної вправи групи. Це composition-level і лишається у superset-config-sheet.html
  • Assumption: Group card has its own drag handle для перетягування групи цілком (на group header). Spec §4.6 каже "drag handle на лівому краю" — діти за рейкою, тож два рівні drag (вся група vs reorder всередині) читаються як ієрархія, а не як один список

Edge cases · related states

  • Без групи — builder.html
  • 2+ групи (B, C) — інлайн letter chip на хедері ротується по visual §2.5 (B = ochre, C = plum, D = sage, E = rust); читається Superset B, Superset C тощо
  • Група розпалась до 1 вправи через Remove (per spec §4.5) — auto-ungroup, картка стає звичайною ex-card
