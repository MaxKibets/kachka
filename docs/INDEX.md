# Documentation Map

Single source of truth for navigating this repo's documentation. Organized by directory, one
sentence per entry stating each file's purpose — enough to decide "is what I need in here?"
without opening the file.

## Root

- `README.md` — placeholder stub, just the title and no real content yet; skip opening it.
  Intended to eventually hold the project overview, prerequisites, and quickstart for
  installing and running the app locally.
- `draft_docs/` — legacy pre-reset draft documentation, kept read-only; consult only to
  clarify intent that `docs/` doesn't cover yet, never authoritative, and out of scope for
  docs-manager to edit.

## `docs/`

- `docs/INDEX.md` — this file; the documentation map itself.
