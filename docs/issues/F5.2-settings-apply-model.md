# F5.2 — Settings sheets: instant-apply vs Cancel/Done conflict

- **Priority:** P1
- **Effort:** S
- **Type:** Spec / logic
- **Source:** design-review 2026-05-23 (F5.2)

## Affected wireframes

- `docs/wireframes/settings-theme-language-sheet.html`

## Problem

Spec says Theme/Language **apply instantly**. But the sheet shows a
`Cancel / Done` footer, which implies a commit step — contradictory. Separately,
the selected row is **double-encoded**: a leading mallard dot **and** a trailing
checkmark; one selection marker is enough.

## Spec reference

`docs/spec/profile.md` §12.2 — "застосовується миттєво" (applies instantly).

## Decisions needed

1. **Apply model.** Pick one and reconcile mockup + spec:
   - *instant* — tap option → apply + dismiss, no footer; or
   - *staged* — keep `Cancel / Done`, drop "instant" from the spec.
2. **Selection marker.** Keep a single marker (dot **or** check, not both).

## Done when

- Sheet and `profile.md` §12.2 agree on one apply model.
- Selected row shows exactly one selection marker.

## Resolution

✅ Decided: **instant apply** + **single marker (checkmark)**. In
`settings-theme-language-sheet.html` the `Cancel / Done` footer was removed —
tapping an option applies immediately and dismisses the sheet, matching
`spec/profile.md` §12.2 ("applies instantly"); no spec change needed. The double
selection marker was reduced to a single trailing checkmark (the leading mallard
dot was dropped); the active row keeps its mallard label tint.
