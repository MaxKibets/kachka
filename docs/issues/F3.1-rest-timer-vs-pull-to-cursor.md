# F3.1 — Rest timer vs pull-to-cursor compete for the bottom zone

- **Priority:** P1
- **Effort:** M
- **Type:** Interaction / layout
- **Source:** design-review 2026-05-23 (F3.1)

## Affected wireframes

- `docs/wireframes/rest-timer.html`
- `docs/wireframes/pull-to-cursor.html`

## Problem

The `rest-timer` mockup uses a large circular ring floating in the content area
above the bottom bar; `pull-to-cursor` puts its chip in nearly the same place.
INDEX already flags the ~92px collision.

## Spec reference

`docs/spec/in-workout.md` §5.1 / §6.6 describe the **bottom bar itself** carrying
the rest state (`A · Rest 2:00`) — a compact countdown, not a big ring.

## Decisions needed

1. **Ring vs compact bottom-bar timer.** The ring is more glanceable from a
   distance (good for the gym) but eats ~180px of vertical space and pushes the
   set list. Choose: prominent ring **or** the spec's compact bottom-bar
   countdown. Record the winner in `in-workout.md`.
2. **Stacking.** If both can be active (resting while scrolled away), define the
   stack order — chip above ring, or chip suppressed during rest. The ring can't
   share the same layer/position as the pull-to-cursor chip.

## Done when

- A single rest-timer presentation is chosen and described in
  `docs/spec/in-workout.md`.
- The two wireframes no longer collide in the bottom zone; coexistence rule is
  documented.
