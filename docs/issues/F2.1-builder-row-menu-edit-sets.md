# F2.1 — Builder row menu missing `Edit sets` and `Add note`

- **Priority:** P1
- **Effort:** M
- **Type:** Functional gap
- **Source:** design-review 2026-05-23 (F2.1)

## Affected wireframes

- `docs/wireframes/builder-row-menu-sheet.html`

## Problem

`builder-row-menu` offers only Add to superset / Move up / Move down / Remove.
The spec also lists **Edit sets** (reps/range/RPE/warmup/add-set) and
**Add note**. Without them there is no pre-workout way to change the default
3×8 or set targets — the user is forced to start the workout to edit structure.
This is a functional gap, not just polish.

## Spec reference

`docs/spec/builder.md` §4.4 (row menu actions). Also flagged as an open question
in `docs/wireframes/INDEX.md`.

Reference: Fitbod exposes per-exercise edit affordances inline — supports adding
`Edit sets` here (`ref-fitbod-builder.png`).

## Action

1. Add **Edit sets** and **Add note** to the builder row menu sheet.
2. Decide whether `Edit sets` opens an inline editor or a dedicated sheet, and
   reconcile that flow with the spec.

## Done when

- Row menu sheet shows Edit sets + Add note alongside existing actions.
- The set-editing entry point is described in `docs/spec/builder.md` §4.4.

## Resolution

✅ Superseded by the Builder / In-workout redesign — see
[`REDESIGN-builder-inworkout.md`](REDESIGN-builder-inworkout.md).

The original fix (add `Edit sets` + `Add note` to the row menu, `Edit sets`
opening a dedicated set-list sheet) was applied, then **revised** by the
redesign:

- The Builder is now **composition-only** — exercises carry no sets there, so a
  set-editing entry point in the row menu is moot. **`Edit sets` was removed**
  from `builder-row-menu-sheet.html`.
- **`Add note` stays** — still a valid per-exercise composition action.
- Set count / reps are now configured in the **In-workout pending state**
  (spec §5.9), not via a Builder sheet. Spec §4.4 updated: the row menu is
  composition-only (`Add note`, `Add to superset`, `Move up / Move down`,
  `Remove`).

Net: the original gap ("no pre-workout way to change the default 3×8") is closed
differently — by the In-workout pending state, where the default is **1** set
and `Add set` copies the previous set.
