# F2.2 — Superset config sheet isn't mode-aware

- **Priority:** P1
- **Effort:** S
- **Type:** Spec / logic
- **Source:** design-review 2026-05-23 (F2.2)

## Affected wireframes

- `docs/wireframes/superset-config-sheet.html`

## Problem

`superset-config-sheet` shows `Cancel · Save · Ungroup` together. **Ungroup**
only makes sense in *edit* mode (a group already exists); in *create* mode there
is nothing to ungroup. The footer must be mode-dependent. Also the spec names
the create action **"Create group"**, but the mockup says "Save".

## Spec reference

`docs/spec/supersets.md` §6.2 (combined config sheet, "Create group" label).

## Action

Make the footer mode-aware:

- **create** → `Cancel` / `Create group`
- **edit** → `Cancel` / `Save` + `Ungroup`

Rename the create button to **Create group** to match the spec.

## Done when

- The sheet renders the correct footer per mode (create vs edit).
- Create-mode button reads "Create group".
- No `Ungroup` action in create mode.

## Resolution

✅ Fixed in `superset-config-sheet.html`. The rendered sheet is now the **create**
mode: footer `Cancel` + `Create group` (renamed from `Save`, matching
`spec/supersets.md` §6.2), with **no** `Ungroup`. The edit-mode footer delta
(`Cancel` + `Save` + `Ungroup`) is documented in the annotations. No spec change
needed — §6.2 already names the action "Create group".
