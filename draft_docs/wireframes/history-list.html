Opens from

  • Tab bar → History з будь-якого таба
  • App launch якщо History був останнім активним табом перед закриттям (стандартне tab persistence)

Opens to

  • Tap на row → history-detail.html (push на History stack)
  • Top-bar без actions у v1 (per spec §10.1–10.2). Filters / search / export → v2 (spec §16)
  • Tap Today tab → today-has-history.html
  • Tap Profile tab → profile.html — TBD

Interactions

  • Vertical scroll. Sticky section header (поточної групи) приклеюється до верху scroll-area при прокрутці (per spec §10.2)
  • Infinite scroll: при наближенні до кінця завантажує наступну сторінку (lazy loading), показує subtle loader внизу. Per spec §10.2
  • Tap row — push detail з smooth horizontal slide (стандарт RN navigation)

Notes

  • Date format per spec §10.2: relative для останніх 7 днів (Today, Yesterday, Mon), absolute далі (28 Apr). Час додано після — task spec показує приклад Tue, Apr 28; обрав компактніший варіант з часом, бо в межах одного дня може бути кілька тренувань (хоча в v1 нечасто — допомагає disambiguate)
  • Section headers — This week, Last week, потім по місяцях (April 2026). Перші два тижні — relative, далі — calendar months. Проста і знайомий патерн (Apple Photos, Material lists)
  • Row layout: 2 inner рядки + chevron справа (≈68px висоти включно з padding). Per visual.md §4 — touch target ≥ 44pt; легко витриманий
  • Volume / sets / duration в одному meta-рядку через separator ·. Plex Mono для чисел, Inter для одиниць і слів — узгоджено з visual §3.2
  • Top-bar ⋯ прибрано (design-review F4.1): у v1 меню нічого не робить — filters / search / export усі в v2 (spec §16). Root tab screen без actions, узгоджено з history-empty

Edge cases · related states

  • Empty (no workouts ever) → history-empty.html (spec §10.4)
  • Picker mode (Choose from history per spec §3.3) → history-picker.html (той самий рендеринг row-ів, але з close-icon, hint subheader і без tab bar)
  • Один workout total — все одно показуємо section header (This week) + одну row, без infinite loader
  • Дуже довга історія (200+ workouts) — same flat list, infinite scroll розв'язує. Без pagination UI
  • Soft-deleted custom exercise всередині workout — у цій row не змінює meta (sets/volume з historical state); назва вправи у detail зберігається через entity ID (per spec §11.9)
